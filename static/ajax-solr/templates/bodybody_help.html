How to Use the AstroExplorer

AstroExplorer aims to be your research portfolio application. It allows you to explore astronomical resources such as publications, observations (and their associated datasets), objects, catalogs, and proposals, and to explore the relationships between them. It then enables you to save these resources into tagged libraries which you can access later, and into collaborations, so that you can share your findings with your co-workers and collaborators.

We expose these resources and their connections by curating and indexing them all at one place. This is the model used by google, where the data resides on your servers, but the index on theirs. We have currently indexed all Chandra Observations, observations from several missions from MAST, related articles, objects, and observing proposals (88K observations and 10K publications). This is part of our current focus on NASA missions, since they have already curated extensive bibliographies. If you own a data set with bibliographic links which you would like to see curated, please contact us at adsvao.astroexplorer@gmail.com.

This is a prototype! It has been created to showcase what kind of functionality can be delivered when these resources are brought together and exposed to the user in a web-based environment. It is meant to provide search and discovery capabilities which are complementary to the position-based searches supported by the VAO portal.

The personalization features which are supported are based on the ADS login system, so they require users to log into their ADS account. This enables one to save queries, resources (publications and observations), tag them, and optionally share them with a group of collaborators.

Script to understand using the application.

First log on, and along the way try and save queries, publications, and datasets. Save results at each point so we have an interesting set of search results to play with.

  1. Describe the main page, displaying publications and facets. Point out the number of publications displayed and the four type of facets available on the left (publications, objects, observations, proposals). Point out that one can sort results in two ways (publication date or citations), ascending or descending.
  2. Type in blazars into search box on publications page. See the 10 autocomplete as keywords but larger set of 146 when you hit ESC to remove the autocomplete and perform at bibliographic search over title, abstract, etc.
  3. See top keywords, active people, most studied CDS objects, targets (mention combining these). Click in left on B1101+384 object, one of the most studied.
  4. choose the one that has chandra mission. Note that it also has data with FUSE.
  5. click on various things associated with the paper to show possibilities. Also note how the left side shows facets of this paper. Notice target normalization is needed. We are tackling it.
  6. Pivot on QSO B1101+384 [NOTE: SIMBAD’s canonical name for this is Mrk 421], notice Schull as main author, and Sembachs on proposals. This is a problem of name normalization that we are tackling
  7. [NOTE: instead of using an object we may want to instead select a target] Facet on mission “Chandra” which reduces the set of results to 27 papers, and continue in observations (317 results)
  8. Show the pop-over under current selection showing all obsids
  9. Select instrument HRC-S which narrows results to 47 observations; sort on exposure time, showing the deepest observations first
  10. Go to Observations tab, type in and autocomplete QSO B1101+384...4361 entries..highly studied
  11. choose stuff taken with IUE/SWP. See FOV, resolution, exposure time facets. Look at Urry, C, Megan proposals, and see authors. Note now that these are not papers, but observations used in papers.
  12. Clear results, and pose the question: what is the impact of one or more observations of a particular target? Pick MAST/Jupiter, click on the “See Metrics” which will display metrics on the 34 papers related to it.
  13. Showcase the use of metrics to evaluate impact of proposals. Start from publications tab, select Chandra, scroll down to Proposal facets and select “Fabian” from the PI facet section. This generates a list of 73 observations (147 publications). Click on “Metrics” to get the aggregate metrics number for all of them.
  14. Show “saved” and collaborations, highlight things that have been saved along the way.
